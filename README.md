## рҹ“Ң Overview

This project predicts the **win probability** of the batting team during the 2nd innings of an IPL T20 match. Given the current match situation вҖ” runs needed, balls remaining, wickets in hand, and run rates вҖ” the model outputs the chance of winning for both teams.

Built using **Logistic Regression** with a full **sklearn Pipeline** for preprocessing and prediction.

## рҹ§  Features Used

| Feature | Description |
|---|---|
| `batting_team` | Team currently batting (chasing) |
| `bowling_team` | Team currently bowling (defending) |
| `city` | Match venue city |
| `runs_left` | Runs still needed to win |
| `balls_left` | Legal deliveries remaining |
| `wickets_remaining` | Wickets still in hand |
| `runs_target` | Total target set by 1st innings |
| `crr` | Current Run Rate |
| `rrr` | Required Run Rate |

---

## вҡҷпёҸ Approach

1. **Data Cleaning** вҖ” filtered to 2nd innings only, removed unknown cities, dropped rows with missing targets
2. **Feature Engineering** вҖ” calculated `runs_left`, `balls_left`, `wickets_remaining`, `crr`, `rrr`, and `result`
3. **Preprocessing** вҖ” OneHotEncoding for categorical features, StandardScaler for numeric features via ColumnTransformer
4. **Modeling** вҖ” Logistic Regression inside an sklearn Pipeline
5. **Prediction UI** вҖ” interactive widget to input live match data and get win probabilities

## рҹ”Қ Key Insights

- Teams with a higher Required Run Rate (RRR > 12) have significantly lower win probability
- Wickets in hand past the 15th over is a strong predictor of match outcome
- Venue (city) has a noticeable effect on win probability due to pitch conditions
